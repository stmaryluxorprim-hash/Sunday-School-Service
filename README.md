# خدمة الكنيسة — Church Service Management PWA

تطبيق ويب تقدمي (PWA) لإدارة خدمات الكنيسة، مبني بمنهجية خطوة بخطوة وقابل للترقية بالكامل.

## نظرة عامة | Project Overview
- **الاسم**: خدمة الكنيسة (Church Service PWA)
- **الهدف**: إدارة المخدومين، الحضور، الإحصائيات، والصلاحيات (RBAC) — بتصميم عربي (مصر) متجاوب، أولوية للموبايل.
- **التقنيات**: Next.js 14 (App Router) · TypeScript · Tailwind CSS · Supabase · PWA
- **اللغة والاتجاه**: العربية (RTL) — خط Cairo

## ✅ المنجز حتى الآن (الخطوة 1: التصميم العام)
- هيكل مشروع Next.js كامل مع TypeScript و Tailwind و PWA manifest + أيقونات.
- نظام تصميم "Aurora" المجرّد الأنيق المتدرّج، عبر متغيرات CSS قابلة للتغيير وقت التشغيل.
- **الهيدر**: أيقونة + اسم الخدمة + الشعار · زر القائمة الجانبية · زر اختيار التاريخ.
- **القائمة الجانبية** (off-canvas drawer) + شريط تنقّل سفلي للصفحات الخمس.
- الصفحات الخمس: الرئيسية · البيانات · الماسح · الاحصائيات · الإعدادات.
- صفحة تسجيل الدخول (تسجيل دخول / حساب جديد).
- **الإعدادات تتحكّم في كل شيء**: اسم الخدمة، الشعار، الأيقونة، نظام الألوان (4 أنظمة)، الوضع الليلي — وتُحفظ محلياً (ولاحقاً في Supabase).

## مسارات التطبيق | Functional Entry URIs
| المسار | الصفحة |
|--------|--------|
| `/` | الرئيسية (Home / لوحة عامة) |
| `/data` | البيانات (إدارة المخدومين) |
| `/scanner` | الماسح (مسح QR للحضور) |
| `/stats` | الاحصائيات (تقارير ورسوم) |
| `/settings` | الإعدادات (تخصيص كامل) |
| `/login` | تسجيل الدخول / حساب جديد |

## معمارية البيانات | Data Architecture
- **مصدر الإعدادات الآن**: `localStorage` عبر `SettingsProvider` (سيُستبدل بـ Supabase).
- **التخزين المخطط له**: Supabase (Auth + Postgres + Storage).
- **RBAC**: كل عنصر تنقّل/تفصيلة له مفتاح صلاحية (`permission`) معرّف في `src/config/navigation.ts` — سيُفعّل بالكامل لاحقاً مع الملفات الشخصية (Profiles).

## ✅ الخطوة 2: المصادقة (Supabase Auth) — مكتملة
- تسجيل دخول / إنشاء حساب فعلي عبر Supabase (بدون تأكيد بريد).
- حماية المسارات عبر middleware: غير المسجّل يُحوَّل إلى `/login` تلقائياً.
- رسائل خطأ بالعربية + حالات تحميل.
- عرض المستخدم الحالي (الاسم + البريد) في القائمة الجانبية + زر خروج فعّال.

## ✅ تحديثات الواجهة والوظائف (الخطوة 3)
- **هيدر ثابت**: الهيدر مثبّت أعلى الشاشة ولا يتحرّك مع تمرير المحتوى.
- **النقاط مباشرة (بدون نافذة)**: إضافة/خصم النقاط من صفحة البيانات تُنفَّذ فوراً
  باستخدام عدّاد النقاط، والسبب يُحدَّد تلقائياً حسب الوظيفة (إضافة/خصم).
- **التواصل**: وظيفة "تواصل" على كل مخدوم تفتح خيارات: اتصال · SMS · واتساب ·
  رسالة داخلية (`src/lib/data/contact.ts` + `contact-sheet.tsx`).
- **تطبيق الرسائل الداخلي**: يُفتح من زر الرسائل في الهيدر (وأيضاً من القائمة
  الجانبية) — قائمة محادثات + شات فوري (Realtime) + إنشاء/حذف المحادثات، ويمكن
  فتح محادثة مرتبطة بمخدوم مباشرة من صفحة البيانات
  (`src/components/messaging/` + `sql/0006_messaging.sql`).
- **الماسح الفعلي**: تشغيل الكاميرا ومسح QR (`html5-qrcode`) مع اختيار الوظيفة
  (تسجيل حضور / إضافة نقاط / خصم نقاط) + عدّاد نقاط + سجلّ كامل لجلسة المسح
  (ناجح/فاشل/الوقت) + إدخال يدوي للكود.
- **الإحصائيات**: صفحة إحصائيات مبدئية حيّة (عدد المخدومين · حضور اليوم · إجمالي
  النقاط · نسبة الحضور · توزيع الجنس · المخدومون حسب الفصل).

## ✅ الإشعارات العامة + إشعارات الجهاز (Web Push)
- **صندوق إرسال في الصفحة الرئيسية**: اكتب أي نص واضغط "إرسال للجميع" ليصل لكل
  المستخدمين.
- **إشعار على مستوى الجهاز (Web Push)**: يظهر الإشعار في إشعارات الجهاز/المتصفح
  (حتى والتطبيق في الخلفية) عبر Service Worker (`public/sw.js`) وبروتوكول VAPID.
- **لوحة إشعارات داخل التطبيق**: زر الجرس في الهيدر (مع عدّاد غير المقروء) يفتح
  لوحة تعرض الإشعارات، وتبقى **غير مقروءة حتى يعلّمها المستخدم كمقروءة** (لكل
  مستخدم على حدة). التحديث فوري عبر Supabase Realtime.
- **الملفات**: `src/components/notifications/*` · `src/lib/notifications/*` ·
  `src/app/api/notifications/send/route.ts` · `sql/0007_notifications.sql`.

### إعداد الإشعارات | Setup
1. شغّل `sql/0007_notifications.sql` في Supabase → SQL Editor.
2. ولّد مفاتيح VAPID مرة واحدة: `npx web-push generate-vapid-keys`.
3. أضف متغيّرات البيئة (في Vercel و/أو `.env.local`):

| Name | ملاحظة |
|------|--------|
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` | المفتاح العام (آمن للمتصفح) |
| `VAPID_PRIVATE_KEY` | المفتاح الخاص (خادم فقط) |
| `VAPID_SUBJECT` | مثل `mailto:admin@example.com` |
| `SUPABASE_SERVICE_ROLE_KEY` | مفتاح service_role (خادم فقط) — للبث لكل الأجهزة |

> عند أول استخدام يطلب التطبيق إذن الإشعارات؛ اضغط "تفعيل إشعارات الجهاز" على كل
> جهاز يريد استقبالها. على iOS يجب تثبيت التطبيق على الشاشة الرئيسية (PWA) أولاً.

## ✅ بوابة المخدوم + بوابة الدخول (Member Portal)
- **بوابة دخول موحّدة `/welcome`**: أول شاشة تسأل «مخدوم أم خادم؟» وتوجّه
  لصفحة الدخول المناسبة.
- **دخول المخدوم بمسح الكارت**: `/member/login` — كاميرا QR (`html5-qrcode`)
  تلتقط كود الكارت وتسجّل الدخول تلقائياً، أو إدخال الكود يدوياً.
- **جلسة موقَّعة (HMAC-SHA256)**: كوكي `member_session` موقّعة httpOnly —
  آمنة ضد التزوير، وتعمل في Edge middleware عبر Web Crypto.
- **تذكرني**: للمخدوم (180 يوماً) وللخادم (كوكي دائمة/جلسة حسب الاختيار —
  عبر كوكي `auth_remember` التي يقرأها الـ middleware).
- **صفحات البوابة** (`src/app/member/(portal)/`): الرئيسية (ملخص) · الحضور
  (سجل كامل) · النقاط (رصيد + سجل) · بياناتي (بيانات + كارت QR رقمي) ·
  الرسائل (شات مع الخدّام) · الإشعارات (مع حالة القراءة لكل مخدوم).
- **طبقة بيانات آمنة**: كل قراءات/كتابات البوابة عبر RPCs بصلاحية
  SECURITY DEFINER مفتاحها كود الكارت السرّي (`sql/0008_member_portal.sql`) —
  المخدوم ليس مستخدم Supabase Auth.
- **قابلة للتوسعة**: أضف صفحة جديدة تحت `src/app/member/(portal)/` وعنصراً في
  `src/config/member-navigation.ts` لتظهر في التنقّل مباشرة.

> إعداد قاعدة البيانات: شغّل `sql/0008_member_portal.sql` في Supabase → SQL
> Editor. اختيارياً أضف `MEMBER_SESSION_SECRET` (سلسلة عشوائية طويلة) في
> متغيّرات البيئة لتوقيع جلسات المخدومين.

### متغيرات البيئة على Vercel | Vercel Environment Variables
أضف هذين المتغيّرين في Vercel → Project → **Settings → Environment Variables**:

| Name | Value | Environments |
|------|-------|--------------|
| `NEXT_PUBLIC_SUPABASE_URL` | رابط مشروع Supabase | Production · Preview · Development |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | مفتاح anon العام | Production · Preview · Development |

> هذه القيم **لا تُحفظ في GitHub** — فقط في Vercel. ملف `.env*` مُستثنى عبر `.gitignore`.
> للتطوير المحلي: انسخ `.env.example` إلى `.env.local` واملأ نفس المفاتيح.

### الخطوات | Steps
1. ادفع الكود إلى GitHub (تم).
2. في Vercel: **New Project → Import** المستودع `Sunday-School-Service`.
3. أضف متغيّري البيئة أعلاه.
4. **Deploy** — سيكتشف Vercel إطار Next.js تلقائياً.
